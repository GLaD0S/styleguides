Sass

All of TTF14's CSS is written in Sass, and partials are imported in a manifest app.scss file. TTF14 uses four Compass imports: opacity, CSS3, transform, and border-radius. It uses responsive, fonts, grid, and tables from tt-layouts. The settings partial sets up color and layout variables. The theme import includes logo styles, general styles such as paragraphs and links, typography, wrappers and containers, and header and footer styles. The pages import includes styles specific to individual pages. The nav import includes menu styles. The mobile menu import includes mobile-specific styles and the off-canvas track menu. A mixins import includes mixins used throughout. There's also an HTML5 boilerplate import.

JavaScript

JavaScript that needs to be loaded includes Modernizr, jQuery, Flexslider, Fastclick, and Offcanvas. Offcanvas is used for the tracks menu.

Branding

The TTF14 site has web- and mobile-specific logos in both the header and the footer. In addition, the background image included in the header and image fixed above the footer are matched to other materials for TTF14 to ensure consistency across. General Texas Tribune colors such as $trib_yellow are used in the TTF14 site for consistency. The unique track colors are also consistent across all appearances of tracks in TTF14 materials to provide a visual cue of the track to which content belongs.

Typography

Fonts

Fonts used in the TTF14 site are proxima-nova, rooney-sans, and rooney-web. These fonts are included with Typekit. There's a font_styles mixin that can be used to set font color, weight, size, and line height, respectively.

Headers

This is the default h1 style.

H1 headers are used mainly for page titles. The default h1 style is color $dark_navy, font weight of 500, font size of 2.25em, line height of 1.2em, 'rooney-sans' font family, a bottom margin of 0.5em, and text aligned left.

This is the default h2 style.

H2 headers are used for session names on the Schedule page and track detail page, speaker names on the track detail and Speakers pages, Sign Up: label in festival subscription, time slot on Schedule page, and Sponsors page headers. The default h2 style is color $light_navy, font weight of 800, font size of 1.65em, line height of 1.2em, letter spacing of 1px, 'proxima-nova' font family, and text transform to uppercase.

This is the default h3 style.

H3 headers are used for 'Back To Top' navigation on mobile, hashtags, session times on Schedule page, and venues on track detail pages. The default h3 style is color $dark_navy, font weight of 500, font size of 1.45em, line height of 1.2em, and 'rooney-sans' font family.

Responsive

Responsive Contexts

Responsive body classes are assigned to pages as the window size changes with responsive-contexts-dev.js. These classes are used in the mobilemenu partial to apply responsive styles.

Grid

The tt_layouts grid is imported and used to apply a grid to the TTF14 site.

Media Queries

The TTF14 site utilizes media queries to ensure that it looks good when viewed at any size. The main media queries used are those for tablet (max-width: 799px) and for mobile (max-width: 520px). Queries are particularly used to ensure that the min height for the header is at least high enough to include the site navs. The mobilemenu import also includes the helper classes .hide-for-mobile, .hide-for-medium-down, .show-for-medium-down, and .mobile-only to display or hide content for medium (tablet) and mobile sizes.

Navigation

Navigation is mainly through the site nav and the pages nav, both included in the header at the top of each page with links to different pages and to social media sites like Twitter, Facebook, and Instagram. Register, Schedule, and Tracks images are included in the right rail and link to pages around the site for ease of navigation. The logo in the footer links to the homepage.

Mobile and Tablet Navigation

At the bottom of the mobile site, Back To Top links to the top of the page. Back To Top has a class of .back-to-top.

Tablet navigation includes the site-nav, but mobile does not. Mobile does include the pages nav menu, with each li adjusted to 50% width. Register, Schedule, and Tracks images, respectively, are also included on tablet and mobile directly below the header.

Each speaker has a headshot image on the Speakers page. If a speaker doesn't yet have an image, a generic speaker avatar image should be used. Images for speakers are entered in Sched.

Paragraphs

Paragraphs with just the p tag are styled with 'rooney-web' font family, color $navy, font weight of 300, font size of 1em, and line height of 1.35em.

Homepage

Paragraphs on the homepage included as part of the 'Updates' section are styled with 'proxima-nova' font family, font weight of 400, and font size of 1em.

Example Code 
						
							<article class="update">
								<p>
									...
								</p>
							</article>

Schedule Page

Paragraphs are used on the Schedule page for session descriptions. These paragraphs are styled with 'proxima-nova' font family, font weight of 400, and line height of 1.2em.

Example Code 
						
							<div class="program-description">
								<p>
									...
								</p>
							</div>

Paragraphs are also used on the schedule page for track headers. These styles are designated with class .track, which floats the track name left.

Example Code 
						
							<div class="program-description">
								<p class="track">
									Track Name
								</p>
							</div>

Venue names are styled with paragraphs with class .venue. This class floats the venue name to the left and gives it 'proxima-nova' font family, color $dark_navy, font weight of 400, and font size of 1em.

Example Code 
						
							<div class="program-description">
								<p class="venue">
									Track Venue
								</p>
							</div>

Speakers Page

The Speakers page uses paragraphs with a class of .panel-link to style the sessions and session names for each speaker. This class applies the styles of 'rooney-sans' font family, font weight of 500, color $dark_navy, font size of 1em, line height of 1.1em, and float left.

Example Code 
						
							<div class="speaker">
								<p class="panel-link">
									Sessions: Session Name, Session Name...
								</p>
							</div>

Lists

List elements are used in the site nav for links, on the Sponsors page for sponsor Twitter handles, on the Homepage for list elements in Updates, in the festival wire for Updates, and on the Schedule page for session speakers.

Sponsors page

The Sponsors page uses list elements within a ul with class .handle_list to style sponsor Twitter handles.

Example Code 
						
							<ul class=.handle_list>
								<li>
									Twitter handle
								</li>
							</ul>

Links

Links are generally styled with color #008990. They have a text decoration of none, a font size of 1em, a line height of 1em, and a hover state of text decoration underline and color #008990.

Default link styling

Schedule Page

On the Schedule page, track names are links. The links are inside paragraphs with class .track. These links have a style of 'proxima-nova' font family, color white, font weight of 800, font size of 1.65em, letter spacing of 1px, and line height of 1.2em. These links have text transform of uppercase. On hover, the color remains white.

Example Code 
						
							<p class=track>
								<a>
									Track Name Link
								</a>
							</p>

Day buttons on the Schedule page are links styled with an anchor link inside an li of class .day-btn. These links have a hover state of background $festive_blue, color $dark_navy, and text decoration none.

Example Code 
						
							<li class=day-btn>
								<a>
									Day
								</a>
							</li>

Site Nav

Links are used in the site nav for navigation to different pages. Style for these links is color $navy, display block, 'proxima-nova' font family, font weight of 600, and text transform to uppercase. The active state is a background of $festive_blue. The hover state is a background color of #9ed8d4, color $navy, and text decoration none.

Example Code 
						
							<nav class="nav site-nav">
								<ul>
									<li>
										<a>
										</a>
									</li>
								</ul>
							</nav>

Social Media

Twitter, Facebook, and Instagram social media icons are included in the header and footer of the page. These are Font Awesome icons. The hover state for the icons is $festive_blue. The icons are placed inside links with class of .social and given a class with their appropriate Font Awesome name.

Example Code 
						
							<a class="social">
								<i class="icon-iconname">
								</i>
							</a>

Twitter Hashtags

The Twitter hashtags for the festival and for each track are included with links that open new tabs to the appropriate Twitter hashtag search page using target="_blank". The festival hashtag of #tribunefest is a link included as an h3 in the site header. The hover state is an underline of the same color. The #tribunefest link is not displayed on mobile.

Example Code 
						
							<div class=nav>
								<h3>
									#tribunefest
								</h3>
							</div>

Each track's hashtag is included on track detail pages. On the track detail page, an h3 with a class of .hashtag is used to style the hashtag. The hover state is an underline of the same color.

Example Code 
						
							<body class=track-detail>
								<header class="track_header">
									<h3 class="hashtag">
										#TTF14-track-name
									</h3>
								</header>
							</body>

Sched

The TTF14 site's Django backend is fed info through Sched's API. In Sched, information about sessions, speakers, speakers' images, and more is entered.
